Introduction

Requires HS Prim version 13 or higher


Since Version 13 one HS Prim can monitor:
  • the current land parcel regardless of size. (Range set to  "ThisLand")
  • all land in the region owned by the current land's owner. Regardless of how or where those parcels are within the region. (Range set to "LandOwner's")
  • the whole region. (Range set to "Region")
Therefore it is unlikely most people will need to use more than one HS Prim. These instructions are for advanced users and wont be needed by most people.


The HS Prim has the ability to be integrated into a network of devices and respond to some commands collectively.

Any HS Prim within 20m of another HS Prim will result in 2 menu windows being sent to the HS operator. I can not fix this, it is the nature of SL dialog menus.

Networking is done by each HS system llRegionSaying to each other on a hidden channel1 which is not seen in normal chat.

1 The channel is set in the 'HomeSecurity Channel' notecard setting called channel=number (where number is an integer number) all devices in the network must be on the same channel. 8 is the default channel.

Each HS Prim can be set to respond to commands that have been passed on from other HS systems or HS relays that are within the same Region. (and within 100m from the Prim across sim borders).

Each HS Prim must be told which devices it should accept these commands from for security reasons. Each object in Second Life has a unique UUID Key and we use this to create a HSID. The HSID is then used to identify which other objects can send commands to this object.

1 chance in 4,294,967,296 of a duplicate HSID network identity. So watch for that! :)

Placing the HS into your inventory and then bringing it back out again will cause its HSID to change!

The command /8hsid will print a HomeSecurity device's HSID so you can copy it from History (Ctrl-H).

Shouting the command /8hsid can also be used to detect other HomeSecurity systems within 100m.

HS Prim version 11.043 onward shows the HSID in the Settings menu. With Viewer 2 you can select and copy it. Good for copying a HSID from a distant HS Prim.

You can use the command /8addn HSID (where HSID is the other Prims HSID key) to add a device that can relay commands to this device. Make sure other HS Prims can't hear your command too!

Alternatively in the network=HSID (where HSID is the other Prims HSID key) section of the Settings notecard you can add the HSID of devices that this HS Prim can recieve commands from BUT this is generally not a good idea unless you intend to reset the Prim to notecard values using the /8reset command.

A More Complex Example (Version 13+)

This example is only useful if you want to protect some land parcels but not others in the same region.

(If you wanted to protect ALL your land parcels in the region you would only need 1 HS Prim placed anywhere in the region, choose the range of "Land Owner's" and ignore the rest of this page.)


In this example from the above picture, three land parcels are protected by networked HS Prims while the fourth parcel is not.


In this set up HS #1 would optionally be set up to listen to HS #2 and #3 and each of #2 and #3 would listen to HS #1. There is no benefit to having all devices listening to all other devices. It is best to have a 'chain link' of listening devices relaying messages.

The HS Prims are able to determine if they have seen a particular command previously and will silently reject a command that comes back to it. This will stop feed back loops in the network.
